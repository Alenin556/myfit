# myfit (My Pro Health Nutrition)

Flutter‑приложение для питания и тренировок: профиль, расчёт суточных калорий и КБЖУ, генерация рациона на день, планы по весу, экспорт в PDF и отправка сводок.

---

## Ключевой функционал

### Вход, регистрация и сессия

- **Экран приветствия** (первый запуск без сессии): «Войти» при уже сохранённом профиле, вход по логину и паролю (пароль хэшируется локально, см. `auth/local_auth.dart`).
- **Онбординг (регистрация):** имя, опциональный ник, логин, пароль, год рождения, вес, рост, пол, **аватар** (загрузка из галереи, хранение в base64 в профиле).
- **Выход** из сессии и **повторный вход** (профиль остаётся в локальном хранилище).

### Профиль

- Просмотр и редактирование данных, **рост** после онбординга фиксирован (логика «закрытия» роста в сценарии профиля).
- История/контекст **веса** (`WeightEntry` в `user_profile`).

### Личные цели

- **Цель питания:** набор мышечной массы, сохранение веса, снижение веса.
- **Цель по весу (набор/снижение):** ввод **шага 2, 4, 6, 8 или 10 кг** и диапазона **«от — до» (кг)** на экране **«Данные для расчёта калорий»** (`bmr_entry_page`); данные в `UserProfile` (`targetWeightChangeKg`, `goalWeightFromKg`, `goalWeightToKg`) и валидация `isWeightGoalRangeValid`.

### Главный экран (дашборд)

- Краткое **приветствие**, **аватар**, переход в профиль, **настройки**, боковое **меню разделов**.
- Блоки **плана питания** и **плана тренировок** (карточки с пояснениями и быстрыми действиями).
- На **широком экране** — боковая **навигация** (в т.ч. **«Что поесть»** первой в списке, когда питание «разблокировано»), на узком — **Drawer** + AppBar по разделам.
- **До подтверждения плана (см. ниже):** в меню показываются **Главная**, единый пункт **«Питание»** (экран плана в режиме заглушки + переход к расчёту), **План тренировок**. После нажатия **«К плану»** на экране **«Данные для расчёта калорий»** (`hasMealPlan` в `user_storage`) — полный набор: **«Что поесть»**, **«Подсчёт калорий»**, **«План питания»** и т.д. (`dashboard_page`, `nutrition_plan_gate.dart`).

### Суточная калорийность (БМР) и «Подсчёт калорий»

- **«Данные для расчёта калорий»:** **рост, возраст, пол, вес, активность, цель питания** и при наборе/снижении веса — **блок «Цель по весу»** (шаг 2–10 кг, «от/до») для расчёта **BMR по Миффлина—Сан Жеора** (`bmr_entry_page`). Кнопка **«К плану»** сохраняет параметры, выставляет `hasMealPlan` и открывает сводку **суточных норм (КБЖУ)**.
- **Доступ к питанию:** меню и контент **«Что поесть»**, **«Подсчёт калорий»** и полноценный **«План питания»** доступны **после** первого подтверждения **«К плану»**; до этого показывается **единый экран-заглушка** с пояснением и кнопкой **«К плану»** (`widgets/nutrition_plan_gate.dart`).
- **«Подсчёт калорий» (дневник питания):** `food_diary_page.dart`
  - пункт **«Подсчёт калорий»** в боковом меню / drawer (после разблокировки плана);
  - **дневник** по дням: записи **сохраняются** в `SharedPreferences` (ключ `food_diary_v1` в `user_storage`, JSON: дата YYYY-MM-DD → список записей);
  - для каждого продукта из `assets/data/products.json` ввод **массы в граммах**; **ккал и КБЖУ** считаются от показателей **на 100 г** (`models/food_product.dart`, `models/food_diary_entry.dart`);
  - **итоги за сегодня** (ккал, белки, жиры, углеводы) и сравнение с **нормой** из плана (`computePlanMacroTargets`) обновляются при изменении списка;
  - **панель «Добавить продукт»** — `showModalBottomSheet`, после добавления **не закрывается автоматически**, чтобы сразу ввести следующий продукт; закрытие — **«Готово»**;
  - при **превышении** суточной нормы по **ккал** или **любому макро** — модальное окно: *«Вы превышаете норму. Уверены, что хотите отойти от плана?»* — **«Нет, отменить»** / **«Да, добавить»**.
- Переход **«Подсчёт калорий»** с экрана **плана питания** (кнопка) открывает тот же `FoodDiaryPage` в стеке навигации.

### План питания

- **Текстовый план** по целям и параметрам: калории, вода, приёмы пищи, КБЖУ (см. `meal_plan_narrative` и сводки на экране).
- **Генерация меню на день** по целевым ккал: распределение по приёмам, подбор продуктов из локального **справочника** `assets/data/products.json`, балансировка к цели (`daily_menu_generator`, `menu_balance`).
- **Обновить план** — смена «семени» выбора блюд без сброса профиля.
- **Экспорт PDF** меню на день (таблица, БЖУ, печать/шаринг) — `menu_pdf_service`, `pdf_export_*`.
- **Отправка сводки** (текст плана) — внешние сценарии **почта / Telegram**; для Telegram используется **HTTP API** к вынесенному **Cloudflare Worker** (`telegram_sender_api`, проект `telegram_worker/`).
- На **плане** для набора/снижения — **цель по весу** вводится в **«Данные для расчёта калорий»**; **пример меню** и **PDF** доступны, когда `isWeightGoalRangeValid` выполняется.

### «Что поесть»

- Список **приёмов пищи** с долей калорий, переход к **подбору продуктов и граммовок** на один приём (`what_to_eat_page`, `what_to_eat_meal_page`); **полный UI** — только при **`hasMealPlan`** (см. заглушку выше).

### План тренировок

- Выбор **типа плана** (зал/дом/сплит/кардио/новичок и т.д.) с **текстовым описанием** программы (`workout_types`, `workout_plan_page`), заметки к плану.

### Настройки

- **Тема:** светлая / тёмная / как в системе (`AppState.themeMode`).

### Данные

- **Локальное хранение** профиля и флагов (`shared_preferences` через `user_storage`): в т.ч. `has_meal_plan`, `food_diary_v1` (дневник), без облачного бэкенда в составе репозитория.

### Тесты

- **Виджет- и навигационные тесты** сценариев приветствия, онбординга, дашборда, настроек (`test/`).

## Обновления (последняя сессия)

### Цель по весу: шаг изменения (набор / снижение)

- В модели профиля поле **`targetWeightChangeKg`** (2, 4, 6, 8, 10 кг), валидатор **`isWeightGoalRangeValid`**, **основной ввод** на экране **«Данные для расчёта калорий»**; подсказки на **плане питания** и **сводка для отправки**; сброс с **`clearWeightGoalRange`**.

### Справочник продуктов

- Расширен **`assets/data/products.json`**; теги совместимы с **`DailyMenuGenerator`**.

### Репозиторий и секреты

- Отдельный Git, **`**/.wrangler/`** в `.gitignore`**.

### Прочее

- Выравнивание полей «От»/«До» на плане питания, **`org.gradle.java.home`** (JDK 17) для Gradle при Java 25+.

## Запуск и тесты

```bash
flutter pub get
flutter run
flutter test
flutter analyze lib
```

### Сборка APK (Android, release)

```bash
flutter build apk --release
```

Готовый файл: `build/app/outputs/flutter-apk/app-release.apk` (и дубликат в `build/app/outputs/apk/release/`). На системах, где `java` по умолчанию **25+**, Gradle может падать с `IllegalArgumentException: 25.x / 26.x` при разборе версии. В `android/gradle.properties` задано **`org.gradle.java.home`** на [Eclipse Temurin 17](https://adoptium.net/) (при необходимости путь обновите после смены каталога JDK). Альтернатива: перед сборкой `JAVA_HOME=C:\path\to\jdk-17`.
